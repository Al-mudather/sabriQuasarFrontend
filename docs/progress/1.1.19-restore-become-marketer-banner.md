# 1.1.19 — Restore "become a marketer" banner on Profile + on-brand redesign

- **Date:** 2026-06-18
- **Status:** ✅ done (backend-detection-verified, build/tsc clean)
- **Tag:** `restore-become-marketer`

## Symptom
A normal user had **no way to join the affiliate team / become a marketer**. The
feature existed before but vanished in the phase-4 redesign.

## Root cause
The component `src/components/MyCourses/afilliateBord.vue` (join-pyramid-program
CTA) was originally mounted on **Profile.vue** (`6e0fd4e^:Profile.vue:14`). The
"rebrand to Royal Indigo + phase-4 page rebuilds" commit (`6e0fd4e`) rewrote
Profile.vue and dropped the mount, orphaning the component (nothing imported it).

Distinct from the registration-code gate: that is `PyramidAffiliate` /
`joinPlatform` (permission to USE the platform). This is `Pyramid` /
`joinPyramidProgram` (becoming a MARKETER with an invite code to earn commission).

## Fix
- **Re-mounted `<AfilliateBord />`** at the top of `Profile.vue`'s stack (its
  original home). Non-marketer → "Request to join" CTA (runs `joinPyramidProgram`);
  existing marketer → "Go to your marketing page".
- **Verified detection live:** `myPyramidAccount` returns `null` cleanly for a
  non-marketer (no error), so the join-vs-go branch is correct.
- **On-brand redesign:** removed the `money.png` raster image; added a megaphone
  line-icon in a translucent rounded badge (accent-tinted), deepened the brand
  gradient (`brand-600 → brand-800`), single flex layout, all theme-token based.
  Added an `isMarketer` computed.
- **i18n:** localized the two hardcoded English `notify` strings ("You are now a
  marketer"; fixed "inter"→"enter" on the registration-code prompt) in both locales.

## Verification
Backend non-marketer response confirmed via API; `tsc` clean, `quasar build` OK
(Profile chunk compiles with the component). The live `joinPyramidProgram` click
is a real state change (becomes a marketer) — left for in-app testing with a
has-code, not-yet-marketer account.
