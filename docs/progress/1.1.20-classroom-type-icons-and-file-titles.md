# 1.1.20 — Classroom: content-type icons + smart file titles

- **Date:** 2026-06-18
- **Status:** ✅ done (verified against real backend modelValue; build/tsc clean)
- **Tag:** `classroom-type-icons-file-titles`

## Enhancements
Two requests for the classroom curriculum rail, both verified against the real
production `modelValue` shapes via temporary console logging.

### 1. Content-type icons
Each sidebar item now shows a Material icon next to its VIDEO/FILE/QUIZ label
(`src/components/classroom/CurriculumItem.vue`): a `kindIcon` computed maps
kind → `smart_display` (video) / `description` (file) / `quiz` (quiz) /
`article`. Video deliberately uses `smart_display`, NOT the `play_circle` the
status column uses for "in progress", so type and status icons stay distinct.

### 2. Smart file titles (and generic-title fallback for video/quiz)
Teachers often leave a content item's title as the literal placeholder
`"File Title"`. Real backend data confirmed:
- **FILE** items carry the real uploaded name in `modelValue.attachment`
  (e.g. `/media/.../High_yield_handbook_psychiatry_questions.pdf`). When the
  title is a generic placeholder we now show the derived name —
  "High yield handbook psychiatry questions" (last path segment, decoded,
  extension dropped, `_`/`-` → spaces).
- **VIDEO** items only carry a UUID stream key (`videoUuid` /
  `videoStreamUrl`) — nothing human-readable — so there is no real name to
  recover. Instead of showing the misleading `"File Title"`, a placeholder
  title falls back to the kind label ("Video lesson" / "Quiz").

Implemented as shared, type-checked helpers in `src/types/classroom/types.ts`
(`fileNameFromAttachment`, `isGenericTitle`, `resolveFileTitle`,
`titleFromModelValue`), replacing the title logic that was copy-pasted in
`useUnitContents.ts` and `useCurrentContent.ts`. Because both the rail and the
content view now go through one `titleFromModelValue`, the sidebar and the
open-lesson heading always agree.

## Backend note
Videos have no human-readable name in the current schema. The only real fix for
video titles is server-side (expose a video name / original filename) — flagged
for the classroom backend spec if pursued later.

## Verification
`tsc --noEmit` clean, `npm run codegen:check` clean, `quasar build -m spa`
succeeded. Temporary `[CLASSROOM-DEBUG]` logging used to confirm the real
modelValue shapes, then removed. Final visual confirmation is in-app (classroom
is auth-gated).
