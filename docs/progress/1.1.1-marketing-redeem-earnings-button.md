# 1.1.1 — Marketing: redeem-earnings button

- **Date:** 2026-06-15
- **Status:** ✅ done
- **Commit:** `d10e95a`

## Goal
The pyramid/marketing money flow has two steps: **redeem** accrued earnings
(ledger reward) into the spendable **balance**, then **withdraw** the balance.
The redesigned `MyMarketingPage.vue` kept "withdrawal request" but dropped the
**redeem** button, so users could see earnings they couldn't move.

## What we built
- Restored the redeem action on
  `src/pages/pyramid_marketing_management/MyMarketingPage.vue`, under the
  "Total earnings" metric.
- Wired the existing `ClaimPyramidLedgerBalance` mutation with
  `refetchQueries` for `MyPyramidLedgerReward` (resets toward 0) and
  `MyPyramidBalance` (increases).
- Added `redeemEarnings()` handler + `claiming` state; reused the page's
  `errorHandler`.
- i18n keys for the button + success toast (ar + en).

## Verification
`tsc --noEmit` clean, `quasar build` succeeds, headless mount of the route OK.
