# 1.1.27 — Classroom video: YouTube-style auto-hiding controls (fullscreen)

**Status:** ✅ shipped (build green; windowed unchanged + the fullscreen hidden/shown CSS states verified live; user-confirmed working in real fullscreen)

Students reported that in fullscreen (especially rotated/landscape) the custom
control bar stays on screen and covers the lesson content.

## Change — `TypeHasifHlsPlayer.vue` (the self-hosted HLS "Al-Hasif" player)
Added YouTube-style auto-hide, **scoped to fullscreen** (the only place the bar
overlays the picture; windowed keeps the bar BELOW the video where it covers
nothing):
- Controls + the mouse cursor **fade out after 2.8s idle while playing**, and
  **reappear on any pointer/touch/key activity**.
- **Never hide while paused**, or while the pointer is **over the control bar**
  (so the user can aim for a button).
- **Touch:** a tap while controls are hidden only **reveals** them; a tap while
  visible toggles play.
- Uses **opacity + transform** (not `display:none`) so controls stay in the DOM
  and remain keyboard/focus reachable — the Video.js/YouTube approach.

Mechanics: a `controlsVisible` flag + a `HIDE_DELAY_MS` idle timer; `@pointermove`
/`@touchstart`/keydown nudge it; `watch(state.playing)` and `watch(isFullscreen)`
drive the cycle; `data-controls-hidden` on the shell toggles the CSS (fade +
`translateY(110%)` + `pointer-events:none` + `cursor:none`).

Researched against [CodeLessGenie](https://www.codelessgenie.com/blog/hide-not-remove-html5-video-controls/)
and the [Video.js controls-visibility blog](https://videojs.org/blog/hiding-and-showing-video-player-controls)
(opacity-based, ~2s idle, hide cursor in fullscreen, tap-to-toggle on touch).

## Scope
Only the HLS/`type_hasif` player. VdoCipher/YouTube use their own iframe controls
(native auto-hide). Vimeo could get the same treatment as a follow-up.

## Verification
- `npx quasar build -m spa` → succeeds.
- Live: windowed player unchanged (controls `opacity:1`, `data-controls-hidden`
  unset); simulating the fullscreen idle state gives the bar `opacity:0` +
  `translateY(110%)` + `pointer-events:none` + shell `cursor:none`, and revealing
  restores `opacity:1`/`pointer-events:auto`. Real fullscreen confirmed by the user.
