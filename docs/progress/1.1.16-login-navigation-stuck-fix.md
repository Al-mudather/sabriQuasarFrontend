# 1.1.16 — Auth: simplify + harden post-login navigation (fix "stuck" login)

- **Date:** 2026-06-16
- **Status:** ✅ done (build/headless-verified; live login confirmation welcome)
- **Commit:** `3b0bd79` · **tag:** `login-navigation-robust`

## Symptom
After a successful login (Google or email) the page sometimes **hung on the
login screen** with no spinner and no navigation, so the user clicked login again.

## Root cause — accumulated redundancy + no timeouts
The post-login path had grown four layers deep:
1. Each login component's `checkRegistrationCode()` fired **3 sequential queries**
   — `CheckPyramidAffiliate`, a pointless `fetchMyMarketingCode()` →
   `setMyMarketingCode('')` race, and `isUserEnrolled()` — using **outdated
   logic**: it only handled a *thrown* "PyramidAffiliate … does not exist" error,
   but the backend now returns a successful `checkPyramidAffiliate: null` (see
   [1.1.12](1.1.12-registration-code-gate.md)), so that branch was dead.
2. The **global router guard** then ran a **4th query** (`verifyPlatformAccess`)
   on the resulting navigation — with **no timeout**.
3. The navigation pushes were **`void`** (not awaited), so the spinner cleared
   while the guard was still pending → looked stuck.

A slow/hung network on any of those left the navigation pending indefinitely.

## Fix
- **Google / email / Facebook** (`GoogleAuthentication.vue`, `Login.vue`,
  `FacebookAuthentication.vue`): replaced `checkRegistrationCode` /
  `redirectAfterLogin` with one `navigateAfterLogin()` — honor `?redirect=`, else
  enrolled → `my-courses` / else → `Home`. It **awaits** the push (spinner now
  tracks the real navigation) and **time-boxes** the enrollment query (3s).
  Registration-code gating is left to the **global guard** (single source of
  truth), removing **2 queries + the dead catch** from each path. Facebook also
  gained the missing `resetPlatformAccess()`.
- **Router guard** (`router/index.js`): race `verifyPlatformAccess` against a
  **5s timeout, fail-open**, so a slow/hung network can never leave a navigation
  pending.

Net: happy path drops from ~4 queries to ~2; nothing on the login path can hang.

## Verification
`tsc` clean, `quasar build` OK, headless login mount renders with no runtime
errors; no leftover `checkRegistrationCode` / `redirectAfterLogin` references.
Takes effect on stc.training after a deploy.
