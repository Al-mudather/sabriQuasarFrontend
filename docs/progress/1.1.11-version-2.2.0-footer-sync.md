# 1.1.11 — Version bump to 2.2.0 + footer-sync clarification

- **Date:** 2026-06-16
- **Status:** ✅ done
- **Commits:** `95fecdf` (→2.1.0), `ec0fa43` (→2.2.0) · **tag:** `version-2.2.0`

## Goal
Bump the release to 2.2.0, and resolve a reported "the footer version is not
synced with package.json" concern.

## Findings (not a code bug)
The footer reads `process.env.APP_VERSION`, injected from `package.json` at
**build time** (`quasar.config.js:13` does `require('./package.json')` once,
exposed via `build.env`, baked by Vite — `AppFooter.vue:118`).

The "not synced" symptom was a **stale process**: the dev server (and any
committed `dist/`) keeps whatever version it was *started* with, because
`quasar.config.js` reads `package.json` once at process start. We saw the live
dev server showing `v2.0.0` while `package.json` was already `2.1.0`. After a
restart it correctly showed `v2.2.0`.

**Conclusion:** the mechanism is correct — the footer follows `package.json` on
every fresh build / dev-server start. Production **rebuilds on deploy** (user
confirmed), so the footer auto-syncs there; nothing else was needed.

## Verification
Restarted dev server → footer renders `v2.2.0`. Fresh `quasar build` contains
the `2.2.0` literal. `tsc` clean, build OK.
