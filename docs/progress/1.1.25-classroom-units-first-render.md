# 1.1.25 — Classroom: render units first, resolve video in background + header/title swap

**Status:** ✅ shipped (build green; live-verified on course 143 and 169 — units render immediately, the first real video resolves and plays, no hang, no console errors)

Refines [1.1.24](1.1.24-classroom-scan-units-no-hang.md). The scan already never
hung, but it ran inside a separate shell page that showed a full-screen skeleton
**before** the curriculum appeared — so on a course with empty leading units the
student stared at a blank skeleton during the lookups. Per product, the correct
flow is: **show ALL units first, then resolve and open the first video in the
background** with the units already on screen.

## Changes
1. **Merged the shell into the content view.** The bare `class/:coursePk` route
   now renders `ClassroomContentView` directly (deleted `ClassroomShell.vue` and
   the now-unused `ClassroomSkeleton.vue`). The curriculum rail renders ALL units
   immediately from the bootstrap (its own skeleton covers the brief bootstrap
   load); the player area shows a 16:9 skeleton while the first lesson resolves.
2. **First-lesson resolution moved into the content view.** When there's no
   `contentPk`, it walks the units IN ORDER, `await`s `loadUnit` on each, and
   opens the FIRST unit that yields lessons (skips empty leading units; handles
   the "9 empty units, video only in the 10th" case). If nothing is playable
   anywhere → the no-content empty state (rail still visible), never a spinner.
   Re-resolves if the user returns to the bare course URL; cross-course race
   guard (`bootstrap.coursePk === route coursePk`) retained.
3. **Header/title swap (requested):** the classroom header now shows the COURSE
   name (centered); the selected LESSON name is shown above the video inside the
   content view (reverses the 1.1.21 arrangement).
