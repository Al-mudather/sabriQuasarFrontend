# 1.1.8 — Course: post-login redirect + clean `+`-slug URLs

- **Date:** 2026-06-16
- **Status:** 🟡 partial (instructor URLs verified; the Google-login redirect needs a live OAuth test)
- **Commit:** `ee21b42`

## Goal
Two related problems reported on the enrol flow:
1. **Redirect** — an unauthenticated user clicks "Enroll" on a course → sent to
   login → after login should land **back on that course**, but didn't.
2. **URL naming** — course URLs showed `…/course/Medical%20terminology/…` and the
   redirect param double-encoded to `%2520` ("random jargon"). Wanted clean
   `Medical+terminology`.

## Root cause (redirect)
The login page is **Google-first**. The email/password path in `Login.vue`
honored `?redirect=` (via `redirectAfterLogin()` before `checkRegistrationCode()`),
but `GoogleAuthentication.vue` **ignored it** and always pushed `my-courses`/`Home`.

## What we built
- `src/components/Account/GoogleAuthentication.vue` — `checkRegistrationCode()`
  now reads `route.query.redirect` first and pushes it if present (mirrors the
  email path); the registration-code / my-courses fallback runs only when there's
  no redirect.
- `src/utils/courseSlug.ts` (new) — extracted the transliterate + `+`-slug logic
  out of `CourseDetails.vue` (from [1.1.2](1.1.2-courses-short-share-links.md))
  into a shared `courseSlug(title)` util.
- `src/components/shared/CourseCard.vue` + `src/components/Home/SliderItem.vue` —
  use `courseSlug()` for the `:name` route param, so internal links read
  `Medical+terminology`. `CourseDetails.vue` now imports the shared util.

The `:name` segment is cosmetic (`loadCourse` resolves off `:pk`/`:id`), so this
is safe and also cleans the `?redirect=` param.

## Orchestration
