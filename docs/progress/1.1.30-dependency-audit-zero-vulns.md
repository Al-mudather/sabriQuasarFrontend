## The app-vite v1→v2 migration (the hard part)

- **Node gate:** app-vite 2.6.2 hard-requires **Node ≥22.22.0** (a runtime check
  stricter than its `^22` engines field). Installed **v22.23.1** via nvm, set as
  default, bumped `engines.node`, and pinned **`.nvmrc` → 22**.
- **ESM config:** app-vite v2 compiles `quasar.config.js` as ESM, so CJS
  `require('quasar/wrappers')` / local requires break. Converted to `import`,
  inlined the dotenv + `package.json`-version logic, and **deleted the now-dead
  `src/boot/envparser.js`**.
- **Options API:** v2 tree-shakes Vue's Options API off by default — set
  **`build.vueOptionsAPI: true`** so the many Options-API pages don't render blank.
- **unhead v3:** `createHead` moved to the `/client` entry (`useHead` stays root).
