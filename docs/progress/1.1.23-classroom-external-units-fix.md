# 1.1.23 — Classroom: support EXTERNAL units (fixes courses that hung forever)

**Status:** ✅ shipped (codegen synced · `tsc --noEmit` clean · `quasar build` succeeds · live-verified against the real backend on Neurology / course 169: redirects to the first lesson and renders all 7 lessons, no console errors)

## Symptom
Several courses (Neurology = course 169 the reported case) **hung on an infinite
loading spinner** in the new classroom and "didn't work at all", while the **old
classroom (CourseClassRoom2) played them fine**.

## Root cause (found by diffing old vs new + querying the live backend)
A course unit can be an **external reference**: `isExternal: true`. Such a unit
carries **no content of its own** — its lessons live on a *linked* `external`
unit. Neurology's only unit (pk **1333**, `isExternal: true`) has
`ownContents: 0` and borrows its **7 videos from unit 41** via
`external.courseunitcontentSet` (verified live: `external.pk = 41`,
`totalCount = 7`, lessons pk 560–566 — exactly matching `enrollment.progress.total = 7`).

The old classroom + the admin dashboard resolve content through the `external`
relation. The new classroom only ever read the unit's **own** content
(`allCourseUnitContentsByCourseUnit(courseUnitId: 1333)` and the nested
`courseunitcontentSet`) — both empty for an external unit. So:
- `bootstrap.totalContents` came from `enrollment.progress.total` (7), so the
  "no curriculum" empty state never fired, **and**
- the shell waited forever for a first lesson that the (empty) unit never yielded
  → **infinite spinner**.

The migration to the new classroom simply **dropped external-unit support**. This
affects every course built from external units.

## Fix (frontend only, no backend change)
1. `GetCourseByIDSlim`: also select `isExternal` and
   `external { id pk courseunitcontentSet { totalCount } }` per unit.
2. `useCourseBootstrap`: when a unit `isExternal`, build its `CurriculumUnit`
   from the **external** unit — use `external.pk` as the unit pk (the unit that
   actually owns the content) and `external.courseunitcontentSet.totalCount` as
   the lesson count; keep the course unit's own `title`/`order`. Everything
   downstream (rail expand, `loadUnit`, the shell's first-lesson redirect, and
   `useCurrentContent`'s `courseUnit.pk`) then keys on the content-owning unit pk
   consistently — no other change needed.

A normal (non-external) unit is unaffected: it resolves to its own pk + count.

## Verification
- `npm run codegen:check` → in sync · `npx tsc --noEmit -p tsconfig.json` → clean
- `npx quasar build -m spa` → succeeds
- Live (driven browser, real backend, course 169): `class/169` now redirects to
  `content/560` and the rail renders all 7 lessons (L1–L6 + file); no console
  errors. (Video itself shows the DRM "Protected Content" overlay only in the
  headless preview, which lacks Widevine — unrelated to this fix.)

## Note
This also makes the empty/contentless edge case safer, but the real win is that
external-unit courses now load their lessons instead of hanging.
