# 1.1.2 — Courses: short share links

- **Date:** 2026-06-15
- **Status:** ✅ done
- **Commit:** `11c1ccc`

## Goal
Course share links were enormous (~600 chars): the route bakes the course
**title** into the path (`course/:name/:code/:pk/:id`), and an Arabic title ran
through `encodeURIComponent` became a wall of `%D8%A7%D9%84…` (~9 chars/letter).

## What we built
- Rewrote `titleSlug()` in `src/pages/course_management/CourseDetails.vue`:
  transliterates Arabic → Latin, keeps `[A-Za-z0-9]`, collapses other runs to
  `+`, and falls back to `course` when nothing Latin remains.
- Applied the same slug to the SEO `og:url` (previously injected the raw,
  unencoded title — also a latent bug).

## Why it's safe
The `:name` segment is **purely cosmetic** — `loadCourse()` resolves off
`:code/:pk/:id` and never reads `params.name`. So old long links already shared
(Telegram, etc.) still resolve; only the visible slug changed.

## Examples
- `MRCS PART A COURSE` → `MRCS+PART+A+COURSE`
- `مختبرات - الكورس التحضيري…` → `mkhtbrat+alkwrs+althdyry+…`

## Verification
Slug unit-tested against real titles, `quasar build` succeeds, headless mount OK.

## Notes
Transliteration is romanization, not translation (`الكورس` → `alkwrs`), and is
one-directional/cosmetic — imperfect output can't break resolution.
