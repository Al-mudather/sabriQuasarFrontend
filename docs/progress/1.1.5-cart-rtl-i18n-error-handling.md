# 1.1.5 — Cart: RTL/i18n + receipt-upload error handling

- **Date:** 2026-06-16
- **Status:** 🟡 partial (logic/build verified; failure toasts not yet live-verified)
- **Commit:** `1c5ea4c`

## Goal
Two problems on `/cart/payment`:
1. **Design** — the page stayed Arabic/RTL even in English (stepper, uploader
   text, arrows), so English users saw mixed/mirrored UI.
2. **Bug** — users uploaded a receipt, clicked **Send**, and "nothing happened"
   (a silently-swallowed failure).

## How it was run (orchestration)
Two `sonnet` sub-agents in parallel, then orchestrator integration:
- **Agent 1 (design, read-write):** fixed the RTL/i18n.
- **Agent 2 (research, read-only):** diagnosed the silent bug + mined the old
  error handling from git history (`6b3af6c`, `140f7d2`) and produced a failure
  catalogue. (The old logic lived in this repo's history, not the sibling repos.)
- **Orchestrator (me):** implemented the error handling in `bankakPay.vue`.

## What we built
**Design (Agent 1):**
- `src/pages/ShoppingCart.vue` — stepper labels were hardcoded Arabic and the
  `dir` binding was `rtl:rtl`; now translated + flows LTR in English.
- `src/components/utils/FileUploader.vue` — dropzone prompt/hint/buttons/reject
  toasts i18n'd.
- `src/components/ShoppingCard/payment.vue` — back-arrows mirror by direction;
  remaining notify strings wrapped.
- Runtime `dir` already flips via `useAppLocale → syncDom()` (no change needed).

**Error handling (orchestrator) in `src/components/ShoppingCard/bankakPay.vue`:**
- The silent path: the upload mutation's `else` branch reset the spinner with no
  message and never read `dataObj.errors`. Fixed.
- `errorHandler` now **returns a count** of messages shown (and handles bare
  `{message}` / string shapes), so every failure path (order creation, upload
  rejection, unexpected shape) **guarantees** a fallback toast — Send can never
  be silent again.
- Added the empty-cart guard and the `PyramidAffiliate` → referral-code redirect
  (checking the payload errors, fixing the old code's wrong-array bug).
- All toast strings i18n'd, incl. the previously untranslated helper line.

## Verification
`quasar build` succeeds, `tsc` clean, i18n de-duplicated. Failure toasts are
logic/build-verified but only fire on a real backend rejection — **live
confirmation still pending** (see roadmap). Success path indirectly confirmed
