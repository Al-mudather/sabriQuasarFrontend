# 1.1.9 — Course: collapsible instructor bios (show more/less)

- **Date:** 2026-06-16
- **Status:** ✅ done (headless-verified)
- **Commit:** `93b6241`

## Goal
Instructor `qualification` bios on the course page can be a massive wall of text
(hundreds of words) that dominates the layout. Show the first ~5 lines, with a
toggle to expand/collapse.

## What we built
`src/components/courseDetails/courseInstructors.vue`:
- Clamp each bio to 5 lines (`-webkit-line-clamp`), with a per-instructor
  **Show more / Show less** toggle.
- The toggle appears **only when the text actually overflows** — measured via DOM
  `scrollHeight > clientHeight` per paragraph (template-ref map + `nextTick`),
  not a character-count guess.
- State is **per instructor** (a `Set` of expanded node ids), so one long bio
  doesn't expand the others.
- Accessible `<button>` with `aria-expanded`; RTL-correct; indigo text-link style.
- Reuses existing `عرض المزيد` / `عرض أقل` i18n keys (no duplicates added).

## Orchestration & the regression we caught
Ran as a `sonnet` sub-agent (parallel with [1.1.8](1.1.8-course-redirect-and-clean-urls.md)).
The agent's script rewrite **dropped `displayName` + `getImageUrl`** (still used
by the template), which crashed the render into a **permanent loading skeleton**.
`tsc`/`build` did NOT catch it — `.vue` template bindings aren't typechecked.
A **headless mount test** surfaced `getImageUrl is not a function`; both functions
were restored and the page re-verified.

**Lesson (now standing practice):** for any `.vue` change, run the headless
mount smoke test — green `tsc`/`build` is not sufficient for template/runtime
correctness.

## Verification
`tsc` clean, `quasar build` succeeds, and a **headless load of a real course page**
confirms instructors render (names, bios, and the toggle on long bios).
