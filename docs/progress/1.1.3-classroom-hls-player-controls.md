# 1.1.3 — Classroom: HLS player control bar

- **Date:** 2026-06-16
- **Status:** ✅ done
- **Commit:** `d9abcf3`

## Goal
Replace video.js's overlaid, auto-hiding control bar on the self-hosted HLS
player with a clear, **always-visible YouTube-style bar below the video**: a
watched/buffered/remaining scrub with a live %, large play/pause, skip ±10s,
volume, speed, quality, fullscreen. Scope: the `type_hasif` (self-hosted HLS)
player only.

## Architecture (the reusable pattern)
Keep **video.js as the engine** (HLS + ClearKey DRM + quality levels) but run it
with `controls: false` and drive it from custom Vue UI:

1. `src/composables/classroom/useVideoJsPlayer.ts` — reactive bridge over the
   video.js instance. Exposes media state (`playing`, `currentTime`, `duration`,
   `bufferedEnd`, `volume`, `rate`, quality list…) + intent methods
   (`togglePlay`, `seek`, `skip`, `setRate`, `setVolume`, `toggleMute`,
   `setQuality`). Re-emits `begin`/`progress`/`complete`/`error` so the
   `StartLearningUnit`/progress hooks keep working.
2. `src/components/classroom/player/ClassroomVideoControls.vue` — **presentational**
   bar (props in, intents out, no engine knowledge → reusable across providers).
   Secondary controls collapse into a ⋮ overflow menu via a **container query**
   on the bar's own width. Scrub pinned `dir="ltr"` for RTL correctness.
3. `src/components/classroom/player/TypeHasifHlsPlayer.vue` — thin shell: video
   stage + controls below, owns fullscreen (Fullscreen API on the shell, so
   controls stay visible) and keyboard shortcuts (Space, ←/→, ↑/↓, F, M).

`ClassroomContentView.vue` relaxes the forced 16:9 clip box **only for
`type_hasif`** (via `[data-provider]`) so the bar can render below; other
providers keep the clipped box.

## Verification
`quasar build` succeeds, composable `tsc` clean, headless mount OK. **Not**
live-clicked (classroom is auth-gated).
