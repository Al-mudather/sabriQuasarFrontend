# 1.1.26 — Course detail: markdown + read-more, reorder, drop redundant hero teaser

**Status:** ✅ shipped (build + `tsc --noEmit` green; headless-smoke of the Emergency course page: correct order, markdown rendered, read-more toggles, no Vue warnings)

The course-detail page truncated long sections with no way to expand, and the
hero showed a hard-cut 280-char copy of the same description.

## Changes
- **Reusable `ReadMore.vue`** — clamps any slotted content to a max-height with a
  soft fade and an "عرض المزيد / أقل" toggle that appears only when content
  overflows (a ResizeObserver re-measures once async data loads). Generalizes the
  description's existing clamp UX to arbitrary block content.
- **`renderMarkdown`** — extracted a generic markdown → sanitized-HTML renderer
  (marked + DOMPurify) from `renderQaContent`; `renderQaContent` now calls it
  after stripping legacy `<font>` noise.
- **"What I will learn"** (`whatIwillLearn.vue`): each outcome rendered as
  **markdown** (lists/bold/emoji) and the list wrapped in `<ReadMore>`. Loads the
  full set in one page and drops the separate paginate button so there's a single
  "عرض المزيد" affordance.
- **Prerequisites** (`coursePreRequisites.vue`): same — markdown + `<ReadMore>`.
- **Reorder** (`CourseDetails.vue`): the **الوصف (description)** section now
  renders **before** "what you will learn".
- **Hero teaser removed** (`CourseHero.vue`): deleted the `truncatedBrief`
  280-char hard-cut summary (no read-more) — it duplicated the full description
  that now lives, with read-more, in the الوصف section. Also removed its unused
  computed/styles.
- **Hero gap tightened**: with the teaser gone, the hero's 40px bottom padding was
  pure leftover; inspected the live gap (88px = 40 hero pad + 40 column gap + 8
  heading pad) and set the hero `padding-block-end` to 0 → gap now 48px (the normal
  section rhythm).

## Note
Markdown only *formats* what's authored as markdown. A course whose "what I will
learn" is one inline blob (no line breaks) renders as clean prose but won't turn
inline "1. 2. 3." into a list — that's content authoring, not the renderer.

## Verification
- `npx quasar build -m spa` → succeeds · `npx tsc --noEmit` → clean
- Headless DOM smoke (course 197): section order `about → what-you-will-learn → …`,
  `cd-md` markdown blocks present, 5 `rm__toggle` read-more controls, no Vue warnings.
- Live-inspected hero→description gap reduced 88px → 48px.
