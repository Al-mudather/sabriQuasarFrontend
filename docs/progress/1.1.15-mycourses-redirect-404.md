# 1.1.15 — Fix: my-courses post-login 404 (route name vs path)

- **Date:** 2026-06-16
- **Status:** ✅ done
- **Commit:** `db860bb` · **tag:** `mycourses-redirect-path`

## Goal
After a successful login a user was dropped on `stc.training/#/my-courses` → a
**404 "Page not found"** instead of their courses.

## Root cause
`MyCourses.vue`'s unauthenticated `onMounted` guard redirected to login with
`query: { redirect: '/my-courses' }`. But `my-courses` is the route **name** —
the route **path** is `/myCourses` (`routes.js`). After login the redirect logic
pushes the `?redirect=` value as a **path** (`router.push('/my-courses')`), which
matches no route → the 404 catch-all.

## Fix
`MyCourses.vue:236` — `redirect: '/my-courses'` → `redirect: '/myCourses'`.

Audited the whole codebase: this was the **only** wrong path string. Every other
reference uses `{ name: 'my-courses' }` (the name) or `/myCourses` (the path),
both correct.

## Verification
`quasar build` OK; grep confirms zero remaining `/my-courses` path strings. Takes
effect on stc.training after a deploy.
