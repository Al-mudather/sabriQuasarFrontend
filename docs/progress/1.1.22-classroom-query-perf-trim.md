# 1.1.22 — Classroom query performance (field trims, no refetch storm, ring removed)

**Status:** ✅ shipped (codegen synced · `tsc --noEmit` clean · `quasar build` succeeds · live-smoke in a driven browser: classroom mounts, checkmarks render, Q&A loads, zero console errors)

Students reported the classroom (incl. production) is slow to enter and sluggish
when moving between lessons. A full audit of every classroom GraphQL operation
(see the query inventory below) found three causes: heavy unused fields shipped on
every payload, the unbounded course-wide progress list **refetched on every lesson
change**, and an always-`cache-and-network` Q&A query. All fixes are **frontend
only** — no backend change. Every query touched was verified classroom-exclusive
(no other page imports them), so they were trimmed in place.

## The classroom query map (for reference)
Critical path (blocks the spinner): `GetCourseByIDSlim` (`course`), `GetEnrollmentByCourseForCurrentUser` (`enrollmentByCourseForCurrentUser`), `GetCourseUnitContent` (`courseUnitContent`, network-only — VdoCipher OTP). Reactive: `GetCourseUnitContents` (`allCourseUnitContentsByCourseUnit`, lazy per unit), `GetAllLearningProgressByCourse` (`learningProgressByCourse`). Q&A (lazy, on tab open): `AllQuestionsByCourse` (`allQuestionsByCourse`) + `QuestionAndAnswer` subscription (`questionAnswerSubscription`). Writes: `StartLearningUnit`/`EndLearningUnit`.

## Phase A — drop dead fields (`classroom-query-trim-fields`)
Removed fields nothing in the UI reads:
- `GetCourseByIDSlim`: dropped `cover` (mapped but never read in the classroom).
- `AllQuestionsByCourse`: dropped the entire `questionreplySet.edges { node … user }`
  subtree — only the reply **count** is shown; real replies load lazily via
  `AllQuestionRepliesForQuestion` on expand. Kept `questionreplySet.totalCount`.
- `QuestionAndAnswer` subscription: dropped the unused `notification { … }` and
  `answer { … }` blocks (the merge only reads `question`); trimmed its
  `questionreplySet` to `totalCount`. Removed the now-dangling `QnaSubscriptionAnswer`
  type alias.
- `StartLearningUnit` / `EndLearningUnit`: dropped `errors` + unread `learning` fields
  (kept `learning { pk }` on Start; `learning { pk complete }` on End — see Phase B).

## Phase B — stop redundant network work (`classroom-query-no-refetch-storm`)
- **Progress no longer refetches the whole list on every lesson.** `useLearningProgress`
  previously called `refetch()` of the **unbounded** `GetAllLearningProgressByCourse`
  after every `EndLearningUnit` (i.e. on nearly every navigation). Now it patches only
  the single row the student touched, using the row the Start/End mutations return,
  via a session-local override merged over the once-fetched server map. Checkmarks /
  in-progress dots update instantly; a full reload still reconciles from the backend.
  Completion is still recorded (Start/End writes unchanged) so the My Courses % and
  certificates keep working.
- `useCourseQuestions`: `cache-and-network` → `cache-first`. The Q&A panel is already
  lazy (slot is `v-if`-gated by the active tab), so this stops the always-on network
  re-fetch when a user re-enters with Q&A as their persisted tab. The live subscription
  + explicit `refetch()`/`fetchMore()` keep it fresh.

## Phase C — remove the header progress ring (`classroom-header-drop-ring`)
- Per the product decision, removed the `%` ring from the classroom header (kept the
  centered lesson title + exit). Per-lesson completion stays visible as the rail
  checkmarks; the overview panel still shows the numeric progress.
- Removed the per-lesson `refreshEnrollment()` refetch in `ClassroomLayout` — it existed
  only to live-update the now-removed ring (one fewer network call per lesson switch).

## Not changed (deliberately)
- `GetAllLearningProgressByCourse` is kept (it powers the checkmarks the user wants
  visible) — only its refetch-on-every-lesson was removed. Bounding it with `first:`
  pagination + a backend lesson `title` field (to drop the heavy `modelValue` from the
  rail) remain as **backend-dependent** follow-ups (Phase D, not done).
