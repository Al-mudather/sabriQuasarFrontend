# 1.1.18 — Auth: fix registration-code gate (it stopped appearing) + dropdown entry + single-click Google

- **Date:** 2026-06-18
- **Status:** ✅ done (backend-contract-verified, 3-agent audited, live-confirmed)
- **Tag:** `regcode-gate-appears`

## Symptom
Users reported the **registration code page never appeared** after sign-in, no
matter the checks. Separately, Google sign-in still needed **two clicks**.

## Root cause — gate failing OPEN on the no-code signal
The gate (`router/index.js` global guard → `pyramid.verifyPlatformAccess()`)
depends on the `CheckPyramidAffiliate` query. Verified **live** against the
backend with a real no-code account:

| Case | Response |
|------|----------|
| **No code** | GraphQL error `"PyramidAffiliate matching query does not exist."` + `data:null` |
| **Has code** | `data.checkPyramidAffiliate` = non-null object, no errors |
| **Unauthenticated** | error `"You do not have permission…"` + `data:null` |

The Apollo client default is `errorPolicy: 'all'`, so those errors **don't
throw** — they arrive in `res.errors`. The 1.1.17 "stuck login" change had made
`verifyPlatformAccess` do `if (res.errors) return true` (fail OPEN on ANY error),
so the no-code error was read as "has access" → the gate **never fired**. Also
confirmed: `socialAuth` does NOT depend on the backend's removed `sessionid`.

## Fix
- **`pyramid.ts` `verifyPlatformAccess`:** decode the real contract —
  `data.checkPyramidAffiliate != null` → access; the `"…does not exist"` error
  (or a clean `data:null`) → **no code → gate**; only a genuinely unexpected
  error/throw fails open (uncached).
- **`RegisterationCode.vue`:** auth-scoped on-mount **no-double-registration**
  guard — an already-registered user is redirected away ("You are already
  registered"); a `joinPlatform` `success:false` now surfaces an error instead
  of a silently vanishing spinner.
- **`AppHeader.vue`:** new **Registration Code** dropdown entry
  (`/account/registerationCode`), authed-only, in the desktop dropdown + mobile
  drawer, below Personal profile.
- **Google single-click** (`boot/socail_auth.js`, `GoogleAuthentication.vue`):
  `warmGoogle()` loads GIS + builds the token client at **boot** (and on mount),
  and `authenticateGoogle` takes a synchronous in-gesture path when ready, so the
  popup opens on the **first** click. Button shows a brief loading state until
  GIS is ready.
- i18n: added `"أنت مسجّل بالفعل"` (both locales); `"كود التسجيل"` EN →
  "Registration code".

## Verification
Backend contract captured live; 3 parallel audit agents (scenario trace,
adversarial diff review, dropdown/i18n wiring) — all 10 gate scenarios correct,
no redirect loops; their findings (leftover debug logs, `success:false` silent
failure, redundant query) all fixed. `tsc` clean, `quasar build` OK, headless
mounts clean, user-confirmed in-app.
