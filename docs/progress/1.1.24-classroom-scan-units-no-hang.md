# 1.1.24 — Classroom: scan units for the first lesson (never hang) + skeletons

**Status:** ✅ shipped (build green; live-verified on the real backend for course 143 *and* 169 — clean entries redirect to the first real lesson, all units render, no console errors)

Follow-up to [1.1.23](1.1.23-classroom-external-units-fix.md). A second hang case
(course 143 "Anatomy (Cardio-Vascular System)") exposed two more issues.

## What was still wrong
1. **External rule too broad.** 1.1.23 sent *every* `isExternal` unit to its
   linked unit. But course 143's unit 963 is `isExternal: true` **and has 4 of
   its own videos** (its external link points at unit 962, which has 1). Using
   the external there would have shown the wrong/fewer lessons.
2. **The shell only loaded the FIRST unit and waited for it forever.** Course
   143's first unit ("1.1 INTRODUCTION", `totalCount: 1`) actually returns **0**
   playable lessons from the content query; the real videos start at unit 1.2.
   The shell hydrated only unit[0], found nothing, and **spun forever** — never
   advancing to the unit that has videos.

## Fixes (frontend only)
1. **`useCourseBootstrap` — own-content-first rule.** A unit uses its OWN content
   when it has any; it only borrows from its `external` unit when its own count
   is 0. Covers both shapes: Neurology (0 own → external 41's 7) and course 143
   unit 963 (4 own → keep own).
2. **`ClassroomShell` — scan forward, never hang.** On entry it now walks the
   units IN ORDER, `await`s `loadUnit` on each, and redirects into the **first
   unit that actually yields lessons** (skipping empty leading units, and
   self-healing the count-says-1-but-fetch-returns-0 mismatch). If no unit
   anywhere has a playable lesson it settles to the "no curriculum" empty state —
   **never an infinite spinner**. A `bootstrap.coursePk === route coursePk` guard
   prevents a cross-course race when switching courses in-app.
3. **Skeletons replace the three-dot spinner.** New `ClassroomSkeleton.vue`
   (rail + 16:9 player + panel placeholders) renders during the shell's resolve
   phase; the content view shows a 16:9 player skeleton while the bootstrap loads.
4. **All units always render** in the rail — including empty ones (course 143's
   "1.1" shows as `0/0`), which is the requested behavior.

## Verification (live, real backend)
- Course **143**: `class/143` → redirects to `content/2454` ("1.2 Gross Anatomy
  of the HEART") — the empty unit 1 is shown but skipped; the HLS video plays;
  the rail lists all units incl. the empty `0/0` one.
- Course **169** (Neurology): still redirects to `content/560` ("L1") via the
  external unit — own-first rule keeps the external mapping for the 0-own case.
- `quasar build -m spa` succeeds; no console errors on either course.
